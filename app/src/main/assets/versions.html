Neue Version installiert

Es wurde eine neue Version installiert. Änderungen weiter unten.

Hinweis

Falls es zu App-Abstürzen kommt, bitte die Absturzberichte in den Einstellungen aktivieren. Das erlaubt mir (hoffentlich) das Problem zu beheben. Die Absturzberichte können auch dauerhaft aktiviert bleiben.

Änderungen

v2.12.0

  • Möglichkeit Fotos vom Gerät zu verwenden
  • Skalieren von Bildern beim Einbinden, um die Berichtsgröße zu reduzieren. Einstellungsdialog beachten.
  • Kleine optische Verbesserungen

v2.11.0

  • Unterstützung für die Ausgabe des Berichts als xlsx (Excel)
  • Update auf API30 (Android 11)
  • Entfernung von "legacy external storage access" (Google Anforderung)
  • E-Mail handling verbessert

v2.10.0

Berichten kann jetzt ein Status (In Arbeit, Unterbrochen, Erledigt, Archiviert) zugewiesen werden und die Berichtsliste danach gefiltert werden.
Berichte werden jetzt in einer Datenbank gespeichert, anstatt in Dateien. Alle vorhanden Berichte werden beim ersten Start mit dieser neuen Version in die Datenbank migriert.

v2.9.0

Berichte können mit Android Boardmitteln geteilt werden. Dadurch können sie z.B. über messenger oder bluetooth versendet werden oder zu cloud storage Anbietern hochgeladen werden. Mit dieser Methode sollten auch weitere e-Mail Apps funktionieren. Um die neue Funktion zu verwenden gibt es ein zusätzliches Teilen Symbol neben dem e-Mail Versand Symbol

v2.8.0

  • Bei Materialien können jetzt auch Kommazahlen verwendet werden.
  • Man kann eine Einheit für Materialien mit angeben

v2.7.0

Im Falle eines App-Updates einmalige Anzeige der Änderungen.

v2.6.0

Unterstützung für kleinere (Smartphone) Displays.

v2.5.0

Verwendung von lokalen Dateien für Logo/Fusszeile und ODF Template

v2.4.0

Unterstützung von Crashlytics. Versendet beim Absturz der App einen Bericht an den Autor. Muss explizit aktiviert werden.
